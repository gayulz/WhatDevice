# Product — WhatDevice

> 자동 생성: `/auto setup` (2026-06-01)

## 이름
**WhatDevice** — 무슨 기기인지 알려주는 도구

## 한 줄 설명
Apple 기기의 내부 식별자(예: `iPhone18,1`)를 입력하면 해당 기기의 정식 모델명, 발매년도, 한글 표기 등을 찾아주는 **정적 웹 가이드 사이트**.

## 해결하는 문제
Apple은 내부 코드(`iPhone17,3`, `iPad13,8` 등)로 기기를 식별한다. 개발자가 크래시 로그·MDM·기기 진단 도구에서 이 식별자를 만났을 때:
- 어떤 모델인지 즉시 알기 어렵다
- 한국어 자료가 부족하다
- 비슷한 계열 기기와 비교하기 어렵다

WhatDevice는 이 갭을 메우는 빠른 참조 사이트다.

## 핵심 기능

### F1: 식별자 검색
- 메인 페이지에서 `iPhone18,1` 또는 `아이폰 16` 입력 → 실시간 substring 검색
- 클라이언트 사이드 검색 (서버 호출 0회), 80ms 디바운스
- 빈 쿼리 → 전체 160개 기기 카드 표시 (탐색 모드)
- URL `?q=` 파라미터 지원 → 외부에서 검색 결과 링크 공유 가능

### F2: 기기 상세 페이지
- 160개 기기 각각의 고유 페이지 (`/device/{slug}.html`)
- 식별자, 정식 모델명, 한글 표기(베스트 에포트), 카테고리, 같은 계열 기기 목록
- SEO 메타태그 (canonical, og:url, twitter card)

### F3: 가이드 페이지
- **식별자 기초**: `iPhone18,1` 같은 표기가 무엇인지 처음 보는 사용자에게 설명
- **로그에서 식별자 찾는 법**: 크래시 로그, MDM 콘솔, 진단 출력에서 식별자를 추출하는 실용 가이드

### F4: 다크모드
- 시스템 설정 자동 감지 + 수동 토글
- 선택 결과 localStorage 영속화
- FOUC(화면 깜빡임) 방지를 위해 인라인 스크립트로 렌더 전 적용

### F5: 반응형
- 모바일 우선 (기본 스타일이 모바일)
- 480px 이하 그리드·폰트 자동 조정

## 타깃 사용자
- **주 사용자**: iOS/Apple 생태계 개발자 — 크래시 로그·MDM 출력에서 식별자를 만나는 사람
- **부 사용자**: 중고기기 거래자, IT 지원 담당자, Apple 기기 마니아

## 사용 시나리오

### S1: 크래시 로그에서 모델 확인
1. Sentry/Firebase Crashlytics 리포트에 `iPhone17,3` 등장
2. WhatDevice 접속 → `iPhone17,3` 입력
3. "iPhone 16 Pro" 확인, 발매년도·계열 정보 확보

### S2: 한국어 자료 검색
1. 영어 자료만 있는 신형 기기 → 한국어 표기 확인 필요
2. WhatDevice에서 식별자 검색 → 한글 표기 페이지 직링크 공유

### S3: 가이드 학습
1. 식별자 표기를 처음 접한 신입 개발자
2. `/guide/identifier-basics` 페이지에서 표기 규칙 학습

## 모드
- **타입**: 정적 사이트 (SSG, Static Site Generation)
- **호스팅**: GitHub Pages (gh-pages 브랜치)
- **런타임**: 없음 — 빌드된 HTML/CSS/JS가 곧 제품
- **업데이트 빈도**: 신규 Apple 기기 발표 시 `data/overrides.json` 보강 → 재빌드 → 푸시

## 현재 단계
- ✅ 코어 기능 완성 (검색, 상세 페이지, 가이드, 정책)
- ✅ 빌드 파이프라인 완성 (`./gradlew run` 1회로 전체 생성)
- ⏳ 배포 미완료 — `SITE_URL` placeholder 교체 + GitHub Pages 활성화 필요
- ⏳ GitHub Actions 비활성 상태 (`build.yml.disabled`)

## 비목표 (Non-Goals)
- 동적 서버 기능 (회원가입, 코멘트 등)
- Apple 외 기기 지원 (안드로이드, Windows 등)
- 식별자 완전 한국어 번역 (best-effort 수준 유지)
- 빌드 도구 다양화 (Java/Gradle 단일 스택 유지)
