# 애플 기기 식별자 변환기 — 실행 로드맵 & 기술 설계

> 개발자 도구 허브의 첫 번째 도구. `iPhone15,2` ↔ `iPhone 14 Pro` 양방향 변환 + 검색.
> 작성 기준: 바닐라 JS + Java 빌드 스크립트 (프레임워크 없음, 서버 없음).

---

## 0. 한 줄 요약

크래시 로그·애널리틱스에 찍히는 애플 기기 식별자(`iPhone15,2`)를 실제 기종명으로
바꿔주는 한국어 검색 도구. 기기마다 정적 HTML 페이지를 만들어 SEO 롱테일을 잡고,
애드센스로 수익화한다. 향후 안드로이드·기타 개발 도구를 붙여 "개발자 도구 허브"로 확장.

## 2. 데이터 (실측 결과)

### 2.1 원본 구조
- 파일: `https://raw.githubusercontent.com/clo4/apple_device_identifiers/main/devices.json`
- 형태: `{ "제품명": "식별자" }` (값은 전부 단일 문자열)
- 총 333개, iPhone 53개
- 예시:
  - `"iPhone 14 Pro": "iPhone15,2"`
  - `"iPhone 15 Pro Max": "iPhone16,2"`

### 2.2 가공이 필요한 부분
1. **역방향 맵 생성**: 원본은 이름→식별자. 우리는 식별자→이름도 필요 → 빌드 시 뒤집어서 둘 다 생성.
2. **슬러그 변환**: URL용. `iPhone15,2` → `iphone15-2` (콤마·공백 제거, 소문자).
3. **카테고리 필터**: MVP는 iPhone만. (키가 `iPhone`으로 시작하는 것만 추출)

### 2.3 갱신 전략 (중요)
- 원본이 16·17 시리즈를 빠뜨릴 수 있음(실제로 현재 누락) → **단일 소스 의존 위험**.
- 대응:
  - MVP: clo4 데이터로 시작 + 누락분은 수동 보정 파일(`overrides.json`) 만들어 병합.
  - 이후: 여러 소스(clo4, adamawolf gist 등)를 병합하는 로직으로 강화.
- 면책 문구 필수: "비공식 참고용 데이터입니다. 출처: clo4/apple_device_identifiers".

### 2.4 thin content 대응 (애드센스 승인용)
- 상세 스펙(칩셋·출시일)이 원본에 없으므로, 각 기기 페이지를 다음으로 채운다:
  - 식별자 ↔ 기종명 (핵심)
  - 같은 세대/시리즈의 관련 기기 링크 (내부 링크 = SEO에도 좋음)
  - 짧은 설명 + 자주 묻는 질문(FAQ) 1~2개
  - (선택) 출시일·칩셋은 `overrides.json`에 직접 채워 점진 보강

## 3. 기술 구성 (전체 그림)

```
[devices.json]  ──┐
[overrides.json]──┤
                  ▼
        ┌──────────────────┐
        │ Java 빌드 스크립트 │  ← 로컬/CI에서 1회 실행 (서버 아님!)
        └──────────────────┘
                  ▼ (생성)
        ┌──────────────────────────────┐
        │ /dist                        │
        │   index.html       (메인+검색) │
        │   data.js          (검색용 JSON)│
        │   device/iphone15-2.html ...  │  ← 기기 수만큼 정적 페이지
        │   sitemap.xml                 │
        └──────────────────────────────┘
                  ▼ (배포)
        [정적 호스팅: GitHub Pages / Netlify / Vercel — 무료]
```

핵심 원칙
- **Java는 "서버"가 아니라 "빌드 도구"**. 로컬에서 한 번 돌려 HTML을 뽑고 끝.
  서버 관리·호스팅 비용 없음.
- 화면 동작(검색·필터)은 **바닐라 JS**가 브라우저에서 처리.
- 내가 모르는 기술 0개. 전부 Java + JS + HTML/CSS.

## 4. 폴더 구조 (제안)

```
device-id-tool/
├── data/
│   ├── devices.json        # clo4 원본 (다운로드)

│   ├── index.html          # 메인 페이지 템플릿
│   └── device.html         # 기기 상세 페이지 템플릿 ({{placeholder}} 포함)
├── src/                    # Java 빌드 스크립트
│   └── BuildSite.java
├── static/                 # 그대로 복사될 정적 파일
│   ├── style.css
│   └── search.js           # 바닐라 JS 검색 로직
└── dist/                   # 빌드 결과물 (이게 배포됨, git ignore 가능)
```

## 5. Java 빌드 스크립트 설계 (의사 코드 수준)

> 실제 코드는 다음 작업 때 같이 짠다. 여기선 "무슨 일을 하는지" 흐름만.

```
1. data/devices.json 읽기            (Jackson 또는 Gson)
2. data/overrides.json 읽어 병합      (누락 기기 보강)
3. iPhone만 필터링
4. 역방향 맵 만들기                   (식별자 → 이름)
5. 각 기기마다:
     - 슬러그 생성 (iPhone15,2 → iphone15-2)
     - templates/device.html 읽어 {{name}}, {{identifier}} 등 치환
     - dist/device/{슬러그}.html 로 저장
6. 검색용 data.js 생성               (전체 목록을 JS 배열로)
7. templates/index.html → dist/index.html
8. sitemap.xml 생성                  (모든 페이지 URL 나열)
9. static/* → dist/ 복사
```

기술 포인트
- 외부 라이브러리 최소화 가능(JSON 파싱만 Jackson/Gson, 나머지는 표준 라이브러리).
- 템플릿 엔진도 필요 없음 — `String.replace()`로 충분.
- **이 스크립트가 곧 네 백엔드 강점을 보여주는 포트폴리오 조각**.

## 6. 페이지 구성

### 6.1 메인 (`index.html`)
- 검색창 1개: 식별자(`iPhone15,2`)든 기종명(`아이폰 14 프로`)이든 입력
- 바닐라 JS가 `data.js`에서 실시간 필터 → 결과 카드 표시
- 결과 클릭 시 해당 기기 상세 페이지로 이동
- 하단: 짧은 설명 + 애드센스 영역

### 6.2 기기 상세 (`device/iphone15-2.html`)
- H1: "iPhone15,2 — iPhone 14 Pro" (검색 키워드를 제목에 그대로)
- 식별자 / 기종명 / 슬러그 / (보강 시) 출시일·칩셋
- 같은 시리즈 관련 기기 링크
- FAQ 1~2개
- 애드센스 영역

## 7. SEO 전략

- **핵심 무기**: 기기별 정적 페이지 → 롱테일 키워드 자동 확보
- 페이지 제목/H1에 검색어 그대로: `iPhone15,2 무슨 기종`, `iPhone15,2 기종 확인`
- 노릴 키워드:
  - 메인: "아이폰 기종 확인", "아이폰 모델 코드", "ios device identifier"
  - 양방향: "아이폰 14 프로 모델 식별자"
  - 롱테일: 각 식별자 페이지가 자동 커버
- 필수 세팅: `sitemap.xml`, 각 페이지 `<title>`·`<meta description>`·OG 태그, 모바일 반응형
- Google Search Console 등록 + sitemap 제출

- ⚠️ 검색창 하나만 있는 빈 사이트는 승인 거절 위험(thin content).

  - 기기 페이지 53개+ 로 콘텐츠 양 확보
  - 가이드 글 2~3개 작성: "애플 기기 식별자란?", "크래시 로그에서 기종 확인하는 법" 등
  - 이용약관·개인정보처리방침·문의 페이지 추가(승인에 유리)
- 콘텐츠가 어느 정도 쌓이고 색인된 뒤 신청.

## 9. 단계별 작업 로드맵

> 본업 있는 상태 기준, 평일 저녁 + 주말로 쪼개서 진행.

### Phase 0 — 준비 (반나절)
- [ ] GitHub 레포 생성, 폴더 구조 잡기
- [ ] clo4 `devices.json` 다운로드해서 `data/`에 저장
- [ ] 도메인 정하기(나중에 구매해도 됨, 우선 무료 서브도메인으로 시작 가능)

### Phase 1 — Java 빌드 스크립트 코어 (1~2일)
- [ ] JSON 파싱 → iPhone 필터 → 역방향 맵 생성
- [ ] 슬러그 생성 로직
- [ ] 기기 상세 HTML 1종 생성 성공 (먼저 1개만 찍어보기)
- [ ] 전체 기기 반복 생성

### Phase 2 — 디자인/템플릿 (1~2일)
- [ ] `index.html`, `device.html` 템플릿 + `style.css`
- [ ] 모바일 반응형 (애드센스·SEO 둘 다 중요)

### Phase 3 — 검색 기능 (1일)
- [ ] `data.js` 생성
- [ ] `search.js` 바닐라 JS 필터링 (식별자/기종명 양방향)

### Phase 4 — SEO + 콘텐츠 (1~2일)
- [ ] sitemap.xml, 메타태그, OG 태그
- [ ] 가이드 글 2~3개, 약관/개인정보/문의 페이지

### Phase 5 — 배포 (반나절)
- [ ] dist 정적 호스팅 배포(GitHub Pages/Netlify/Vercel)
- [ ] Search Console 등록 + sitemap 제출 + 애널리틱스

### Phase 6 — 애드센스 (색인 후 대기)
- [ ] 색인·트래픽 좀 쌓인 뒤 신청

### Phase 7+ — 확장
- [ ] GitHub Actions cron으로 데이터 자동 동기화 + 재빌드 (← 백엔드 강점)
- [ ] 안드로이드 기기 추가 (라이선스 확인 후)
- [ ] 추가 도구(크론 해설기, 더미데이터 등)로 허브화
